# Submission Package QA

## Verify current requirements

Browse the target journal's official Guide for Authors immediately before finalization. Confirm scope, article type, manuscript format, highlights, graphical abstract, figure formats, declarations, data policy and cover-letter requirements.

## Expected components

- One authoritative manuscript DOCX.
- Separate title page when required.
- Highlights in the journal's length/count limits.
- Graphical abstract in the required aspect ratio and formats.
- Separate numbered figures and figure captions.
- Supplementary tables/figures.
- Declaration of competing interest.
- CRediT, funding, acknowledgements and data availability in the manuscript as required.
- Journal-specific cover letter.
- Raw/processed data, analysis code and README when supplied.
- Manifest, QA report and upload checklist.

## Clean-package rules

- Remove old-journal names and EJA/JIA-style filenames.
- Exclude backup, preview, temporary and `before_*` files.
- Verify every manuscript figure/table reference has a corresponding caption/file and no file is unreferenced.
- Verify ZIP contents after compression, normalizing Windows path separators.
- Do not upload the parent working directory.

## AI declaration rule

Never add, remove or rewrite an AI declaration automatically. Check the target journal's current policy, report the status neutrally, and follow the author's explicit instruction.

## Final report

State separately: complete items, missing/blocking items, non-blocking recommendations, journal-scope risk, unverified visual-render risk and system fields the author must enter manually (reviewers, ORCID, classifications and confirmations).
