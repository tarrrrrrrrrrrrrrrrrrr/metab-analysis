# `lwxg` Template Map

Default example project: `C:\Users\唐梓然\Desktop\lwxg`.

Use it as a pattern library, not as a source of conclusions for another project.

## Source examples

- `paper.pdf` and manuscript DOCX files: original and revision-history examples.
- `count.xlsx`: multi-factor agricultural workbook with replicate-level observations.
- `picture/reanalyse_count_python.py`: analysis and plotting implementation to audit before reuse.
- `picture/analysis_ready_data.csv`: tidy-data output pattern.
- `picture/anova_type3_all_traits.csv`, `mixed_model_*`, `tukey_*`, `effect_sizes_*`: statistical-output patterns.
- `picture/figure*.pdf/png/svg`: panel, combined, 2D/3D, network and grouped-bar examples.
- `picture/*audit*`: project, data, model, figure and reviewer-risk report patterns.
- `picture/submission_ready*.docx`: manuscript assembly and version-control caution examples.

## Historical requirements captured from this template

- Python backend.
- CSV statistical outputs and PDF figures, with 600 dpi PNG copies.
- Times New Roman, user-specified palette, black-edged points/bars/boxes and model-supported `a/b/c` letters.
- Correct figure ordering, single-panel and combined exports, aligned heatmap/network overlays and readable 3D observed points.
- Mixed-model robustness, cautious interpretation of IPAR-RUE surfaces, and no causal overclaiming.
- Full data-variable coverage, source-code audit, manuscript/figure consistency and clean submission packaging.

## Portability rule

For a new project, rediscover factors, levels, replicates, variables, stages, target journal, palette and scientific conclusions from that project's files. Never copy `lwxg` treatment counts, results, coefficients, significance letters, captions or biological claims.
